# HIVE — HDI Intelligence & Value Engine

> AI-powered natural language interface for HDI's daily operational data.

---

## Solution Overview

### What Was Built

A three-part pipeline simulating a core slice of HIVE:

| Part | Description | Notebook |
|------|-------------|----------|
| **A — Data Pipeline** | Ingestion, cleaning, validation, and operational summary of `hdi_daily_ops.csv` (1,631 rows, Jan 2022–Jun 2026) with 2 visualizations | `ops_data_cleaning.ipynb`, `ops_data_analysis.ipynb` |
| **B — Forecasting** | 7-day ahead recursive forecast of new Enterpriser registrations using LightGBM with lag & rolling features | `new_enterpriser_prediction.ipynb` |
| **C — LLM Chatbot** | LangGraph ReAct agent (GPT-4o-mini) with SQL and forecasting tools, responding in Bahasa Indonesia | `hive_chatbot.ipynb` |

### Key Design Decisions & Trade-offs

- **LightGBM over ARIMA/Prophet**: Three models were benchmarked — Ridge (baseline), XGBoost, and LightGBM — on a 60-day holdout. LightGBM was selected for its best MAE/RMSE balance while remaining interpretable. The trade-off is that it requires manual feature engineering (lag + rolling statistics) compared to native time-series models, but this gives finer control over business-relevant signals like promo periods and day-of-week effects.

- **Recursive forecasting**: The 7-day forecast is generated step-by-step, feeding each prediction as input to the next. This avoids the need for a separate multi-output model, but prediction error can accumulate over longer horizons.

- **SQLite in-memory for the chatbot**: Operational data is loaded into an in-memory SQLite database at runtime via SQLAlchemy. This keeps the setup dependency-free (no external DB server) while allowing the LLM to issue real SQL queries through LangChain's `SQLDatabaseToolkit`.

- **LangGraph ReAct agent**: Chosen over a simple prompt-chaining approach to allow dynamic tool selection — the agent decides whether to query SQL or trigger the forecasting function based on the user's question.

### Assumptions

- The dataset is treated as synthetic but structurally representative of real HDI operations.
- "Minggu ini" (this week) is interpreted as the last 7 calendar days from the latest date in the dataset.
- Forecasting uses the feature store (pre-computed lag/rolling features) from the last available date as the starting point.
- `top_product_id` is treated as a categorical identifier and excluded from the regression target engineering.

## Business Insights

### Key Findings

1. **Day-of-week is the strongest driver of Enterpriser registrations.** Analysis shows a consistent weekly seasonality, with certain weekdays outperforming others — likely aligned with HDI's event and briefing schedules.

2. **Promo periods (`is_promo_period = 1`) produce a measurable registration spike.** This feature ranks among the top predictors in LightGBM's feature importance, confirming that promotional campaigns have a direct, quantifiable effect on recruitment.

3. **EP sales follow a weekly rhythm with upward trend**, visible in the 7-day rolling average. Short-term dips tend to recover within the same week, suggesting operational resilience at the unit level.

### Recommendations for Key Leaders

- **Schedule major recruitment pushes on historically high-registration weekdays** to amplify organic momentum rather than fight against natural patterns.
- **Plan promo periods strategically** — the model confirms they work. Avoid overlapping multiple promos where attribution becomes unclear.
- **Monitor lag-1 and lag-7 registration counts** as leading indicators. A dip in lag-7 today is a reliable early warning for next week's numbers.

## Setup & Execution

### Prerequisites

```bash
python >= 3.10
```

### Installation

```bash
git clone <your-repo-url>
cd hdi-recruitment

python -m venv .venv
source .venv/bin/activate        # macOS/Linux
# .venv\Scripts\activate         # Windows

pip install -r notebook/notebook-requirements.txt
```

### Environment Variables

Create a `.env` file in the project root:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

### Run Order

Execute notebooks in this order:

```
1. notebook/ops_data_cleaning.ipynb         → generates cleaned CSV
2. notebook/ops_data_analysis.ipynb         → EDA & visualizations
3. notebook/new_enterpriser_prediction.ipynb → trains & saves models
4. notebook/hive_chatbot.ipynb              → runs the HIVE agent
```
